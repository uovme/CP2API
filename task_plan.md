# CP2API 合并实施计划

## 目标
把 CP2API 做成以 Sub2API 面板/计费/用量记录为基础、兼容 CLIProxyAPI 多账号 token 文件和调度语义的统一项目，并推送到 `github.com/uovme/CP2API`。

## 当前策略
- 以 Sub2API 作为项目骨架：保留现成管理面板、用户/API Key、额度消耗、用量统计、支付和网关能力。
- 从 CLIProxyAPI 合入可落地能力：token/auth 文件格式兼容、批量导入、负载均衡/粘性会话语义说明与字段映射。
- 不删除、不覆盖已有用户文件；CP2API 是空仓库，已复制 Sub2API 基底。

## 阶段
- [x] 阶段 1：克隆源仓库并确认 CP2API 状态
- [x] 阶段 2：复制 Sub2API 基底到空 CP2API 仓库
- [x] 阶段 3：梳理源项目关键能力和集成点
- [x] 阶段 4：实现 CLIProxyAPI/Sub2API token 文件导入兼容
- [x] 阶段 5：把导入能力接入管理 API，并保持面板可用
- [x] 阶段 6：验证后端单元测试和前端构建/测试
- [x] 阶段 7：提交并推送到 CP2API

## 风险
- Sub2API 项目很大，完整“所有功能合并”不适合一次性重写；本轮以可运行、可验证的兼容层为目标。
- 外部 OAuth 刷新/额度接口需要真实凭据，自动化测试只能覆盖解析、导入映射、调度选择等离线逻辑。
- 如果涉及删除或覆盖已有文件，必须先请求用户确认。
